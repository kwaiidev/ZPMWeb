About ZPM

ZPM Robotics explores autonomous, decentralized swarm systems for next-generation drones. Each drone operates as an independent node, communicating peer-to-peer to coordinate decisions and allocate tasks dynamically during a mission.

By adapting to local context and available resources, the swarm remains resilient, scalable, and responsive even in uncertain or contested environments.

How?

Peer-to-Peer Communication

ZPM nodes communicate directly over ROS2 using DDS discovery. Each ROS2 node can publish and subscribe to eachothers topics. Allowing pure decentralized communication.

With this, each node can have all the information of the other nodes in the swarm. This allows for desicion making to happen independently.

Fault Tolerance

With the concept of DDS discovery, if a node fails during a mission, there is no harm to the overall mission. The swarm can continue to operate without the failed node.

Desicion Making / AI

Each node is designed to make desicions independently, and be versatile.

This means that each node in the swarm can be used for a different purpose, while still being all in one.

Each node is equipped with the proper hardware to make these desicions, and then the data is used by ROS2 software in order to make desicions independently, ZPM nodes leverage CV models, and pathfinding algorithms to make desicions.

Scalability

ZPM systems are designed with scalability in mind. We've already established that losing a single node in a swarm will not affect the mission, this means that adding a node to the swarm will only increase the capabilities of the swarm.
